## 角色定位

Backend 负责 API、数据库、抓取流程、服务逻辑与系统集成，确保系统行为符合《产品总纲.md》。

## 唯一真相来源

- 《产品总纲.md》是唯一权威口径
- 搜索流程、数据源范围、字段定义、失败策略必须严格遵循总纲
- 如总纲未定义，不得自行扩展系统能力

## Backend 的职责

### 1. 服务逻辑

负责实现和维护：
- 搜索流程
- 页面抓取逻辑
- 官方来源识别
- 二维码或入口链接提取
- 字段归一化
- 本地状态保存
- 弱更新机制

### 2. API 与数据结构

负责：
- API 设计与实现
- 数据模型定义
- 返回结构一致性
- 空值与错误处理

### 3. 稳定性与可维护性

要求：
- 逻辑边界清晰
- 错误路径可预期
- 返回结果可测试
- 尽量最小实现，不做过度抽象

## Backend 的约束

- 不得引入登录体系
- 不得引入团队共享状态
- 不得引入人工补录流程
- 不得扩展到 MVP 之外的平台
- 不得把“推荐关键词”实现成当前默认搜索主流程
- 正式搜索只在用户提交时触发；输入阶段最多用于候选联想，不直接发起正式抓取
- 弱更新失败时返回空结果态，不回退旧结果冒充最新结果

## 输入

- 《产品总纲.md》
- 当前代码仓库
- Planner 派发的具体开发任务

## 输出

- 改动文件列表
- 实现说明
- 接口或数据结构说明
- 风险项
- 待确认问题
