# AI群聊发现器

> 内部 AI 应用官方群聊发现工具，用于帮助调研人员快速找到目标 AI 产品或项目在 GitHub 上公开露出的官方群入口。

## 当前目标

当前阶段不追求大而全，只追求最短路径出结果。

统一口径如下：

- 用户输入关键词或 GitHub repo URL
- 系统调用 GitHub Search API 搜索相关仓库
- 按 GitHub stars 排序返回相关结果
- 从 GitHub 仓库页面中提取群聊二维码或群入口
- 支持的平台优先是：
  - 微信
  - 飞书
  - Discord

## 当前 MVP 范围

### 已支持

| 类型 | 内容 |
|------|------|
| 搜索输入 | 关键词 / 产品名 / GitHub repo URL |
| 主要数据源 | GitHub Search API、GitHub 仓库页、GitHub README |
| 排序方式 | 以 GitHub stars 为主 |
| 结果形态 | 一个仓库一张卡片 |
| 返回字段 | 名称、简介、GitHub stars、创建时间、最近验证时间、GitHub 链接、群状态、二维码 |
| 二维码处理 | 从 GitHub 页面图片中提取，并存入本地 |

### 当前重点问题

| 问题 | 说明 |
|------|------|
| 返回结果太少 | GitHub 搜索召回仍偏窄，且存在匿名 API 限流风险 |
| 提取不出二维码 | GitHub 页面抓取范围过窄，二维码识别与平台判定仍然偏保守 |

### 暂不强调

| 类型 | 说明 |
|------|------|
| 官网抓取 | 当前不作为主链路 |
| 推荐关键词 | 当前不作为核心功能 |
| 复杂推荐算法 | 当前不支持 |
| 钉钉 / Telegram | 暂不支持 |
| 非官方群收录 | 不展示第三方社区入口 |

## 运行方式

### 前端

```bash
cd app/frontend
npm install
npm run dev
```

### 后端

```bash
cd app/backend
.venv\Scripts\activate
uvicorn main:app --port 8000
```

## 当前实现原则

1. 搜索链路尽量简单，先稳定命中 GitHub 结果。
2. 二维码链路尽量简单，先从 GitHub 页面直接拔图。
3. 图片能保存到本地就先保存，不在当前阶段引入复杂验证。
4. 无群也要返回仓库卡片，避免用户误判为“没搜到相关产品”。

## 当前最重要的判断标准

- 输入 `FastGPT` 时，应能返回 FastGPT 及相关仓库卡片。
- 结果排序应明显受 GitHub stars 影响。
- 若 GitHub README 中出现微信群、飞书群、Discord 的二维码或入口，应尽量被抓取并展示。
- 即使没提取到二维码，也不应完全丢失仓库结果。
