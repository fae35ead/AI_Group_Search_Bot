## 四、图片提取与群入口判断全链路

> 本节覆盖：发现图片 → 下载图片 → 三层判定决策 → 平台识别 → 构卡存储的完整链路，对应 `EntryExtractor.extract()` 的实际逻辑。

### 4.1 入口：从 HTML 扫描图片

```
扫描策略（两条路径并行）：

  A. <img> 标签扫描
       ↓
     找所有 <img> 标签
       ↓
     提取 src / data-src / data-canonical-src 等属性
       ↓
     收集周围上下文文本（alt / title / aria-label /
     父级文本 / 前序标题，合计 ≤ 300 字符）
       ↓
     找最近的上级 <a href>（向上最多 4 层）
       ↓
     进入预过滤

  B. <a href="*.png/jpg/webp/..."> 图片链接扫描
       ↓
     找所有指向图片文件的 <a> 链接
       ↓
     进入预过滤
```

### 4.2 预过滤：快速丢弃无关图片

```
预过滤 _should_consider_visual_candidate()：

  信号文本 = 图片URL + 上下文文本 + 入口链接
             全部转小写

  ┌─────────────────────────────────────────────┐
  │  ① 平台关键词命中？                           │
  │     Discord / 飞书 / QQ / 微信 的任意关键词   │
  │     → 有平台信号 ✅                          │
  ├─────────────────────────────────────────────┤
  │  ② 群聊意图命中？                            │
  │     group / community / join / invite /     │
  │     server / qrcode / qr / 官方群 / 入群 /  │
  │     交流群 / 社群 / 扫码 等关键词            │
  │     → 有群聊意图 ✅                          │
  ├─────────────────────────────────────────────┤
  │  ① 或 ② 任一通过 → 进入下载阶段              │
  │  两者都未命中 → ❌ 直接丢弃，不下载          │
  └─────────────────────────────────────────────┘

  说明：这是性能优化手段，不做严格验证。
       真正的严格判定在后续三层决策中完成。
```

### 4.3 下载图片

```
_download_image() 支持三种来源：

  ┌────────────────────────┐
  │  ① HTTP 下载           │  httpx GET，请求头含 User-Agent
  │  ② GitHub Camo URL    │  解析 hex 编码的真实 URL 并下载
  │  ③ base64 data URI   │  直接解码（如 data:image/png;base64,...）
  └────────────────────────┘
          ↓
  文件大小限制：单图 ≤ 5MB
          ↓
  返回: (image_bytes, content_type)
```

### 4.4 三层判定决策（核心逻辑）

> 每一张图片必经以下三层，层1 优先，层3 最宽松。

```
┌─────────────────────────────────────────────────────────────────┐
│                   图片 → 三层判定决策                             │
│                                                                 │
│  输入: image_bytes, image_url, 上下文文本, 最近链接, 最终URL       │
│                                                                 │
│  ───────────────────────────────────────────────────────────   │
│  【尺寸检查】（所有层共用前置）                                    │
│  ┌──────────────────────────────────────────┐                   │
│  │  OpenCV cv2.imdecode() 读取宽/高         │                   │
│  │                                           │                   │
│  │  宽 < 180px OR 高 < 180px  ──────→ ❌ 丢弃 │                   │
│  │  长宽比 > 1.35              ──────→ ❌ 丢弃 │                   │
│  │  （图片严重变形，不是 QR）                   │                   │
│  │                                           │                   │
│  │  ✅ 尺寸检查通过 → 继续三层判定              │                   │
│  └──────────────────────────────────────────┘                   │
│                                                                 │
│  ───────────────────────────────────────────────────────────   │
│  【层1】OpenCV QR 解码成功                                        │
│                                                                 │
│  ┌──────────────────────────────────────────┐                   │
│  │  detector.detectAndDecode(image)          │                   │
│  │  → decoded_text (非空)                    │                   │
│  │  → points (QR 角点，非空)                  │                   │
│  │                                           │                   │
│  │  QR 解码出内容 ✅                           │                   │
│  │  且内容中含平台关键词？                     │                   │
│  │  → decoded_payload 含 "discord"/"feishu"   │                   │
│  │    /"qm.qq.com"/"wechat" 等              │                   │
│  │                                           │                   │
│  │         两者都满足                         │                   │
│  │           ↓                              │                   │
│  │  ✅ → ExtractedGroupCandidate             │                   │
│  │      platform = 识别到的平台               │                   │
│  │      group_type = UNKNOWN                 │                   │
│  │      image_bytes = 保留                    │                   │
│  │      qrcode_verified = True               │                   │
│  │      decoded_payload = 解码内容            │                   │
│  │      entry_url = 最近链接 / 最终URL        │                   │
│  │      退出三层判定（直接通过）               │                   │
│  └──────────────────────────────────────────┘                   │
│                                                                 │
│  ───────────────────────────────────────────────────────────   │
│  【层2】QR 特征检测到，但解码失败                                  │
│                                                                 │
│  │ points 非空，但 decoded_text 为空                            │
│  │           ↓                                                  │
│  │  从 image_url + 上下文文本 再次检测平台                        │
│  │  URL/文本含平台关键词？                                        │
│  │           ↓                                                  │
│  │  含平台关键词 ✅                                               │
│  │           ↓                                                  │
│  │  ✅ → ExtractedGroupCandidate                                │
│  │      platform = 识别到的平台                                   │
│  │      group_type = UNKNOWN                                    │
│  │      qrcode_verified = False（无法解码）                     │
│  │      降级为 Link 候选                                         │
│  │      退出三层判定                                             │
│  │           ↓                                                  │
│  │  不含平台关键词 → ❌ 丢弃                                      │
│                                                                 │
│  ───────────────────────────────────────────────────────────   │
│  【层3】无 QR 特征，仅靠文本判断                                   │
│                                                                 │
│  │ points 为空（无 QR 特征）                                     │
│  │           ↓                                                  │
│  │  从 image_url + 上下文文本 检测平台                            │
│  │  含平台关键词？                                                │
│  │           ↓                                                  │
│  │  含平台关键词 ✅                                               │
│  │           ↓                                                  │

│  │    ① 上下文含群聊意图关键词                                    │

│  │  两者都满足 ✅                                                │
│  │           ↓                                                  │
│  │  ✅ → ExtractedGroupCandidate                                │
```

### 4.5 平台识别方法

> 贯穿层1/层2/层3，每层判定都调用 `_detect_platform()`。
> 匹配顺序：Discord > 飞书 > QQ > 微信。
> 微信放最后，避免 weixin.org 等非群聊域名误匹配。

```
_platform_hints = {{
  Discord: ('discord', 'discord.gg', 'discordapp'),
  飞书:    ('feishu', 'larksuite', 'lark', '飞书'),
  QQ:      ('qm.qq.com', 'qun.qq.com', 'jq.qq.com', 'qq', 'qq群', 'qq官方群'),
  微信:    ('weixin', 'wechat', '微信'),
}}

匹配规则：
  输入: image_url + 上下文文本（拼接后转小写）
  按顺序遍历 PLATFORM_HINTS，第一个命中即返回
  无命中 → 返回 None（图片丢弃）
```

### 4.6 群类型识别（当前版本）

```
group_type 当前全部为 UNKNOWN。

原因：群类型（交流群 / 答疑群 / 售后群 / 招募内测群）
依赖上下文语义分析，MVP 阶段未实现。
后续可扩展：根据周围文本关键词推断群类型。
```

### 4.7 候选归并与去重

> 从每张图片/链接提取的候选，进入 `_dedupe_groups()` 做去重。

```
_dedupe_groups()：

  ① 仅保留 4 个支持平台：微信 / QQ / 飞书 / Discord
  ② 签名去重（三个优先级）：
     优先A: platform + image SHA1 → 同一图片只保留1条
     优先B: platform + 标准化链接 → 同一链接只保留1条
     优先C: platform + 图片URL  → 保底
  ③ 合并多来源 URL：
     保留最长 URL 列表（source_urls），
     优先填充 entry_url / fallback_url / image_bytes
  ④ 返回: list[ExtractedGroupCandidate]
```

### 4.8 存储与构卡

```
_build_product_card() + _save_qr_code()：

  对每个通过的候选：

  ┌──────────────────────────────────────────────┐
  │  有 image_bytes → 存储到本地                 │
  │    文件名: {repo}_{platform}_{sha1[:8]}.{ext} │
  │    路径: app/backend/data/public/qrcodes/     │
  │    返回: /assets/qrcodes/{filename}          │
  │    → entry = QRCodeEntry(type="qrcode")      │
  ├──────────────────────────────────────────────┤
  │  无 image_bytes 但有 entry_url               │
  │    → entry = LinkEntry(type="link",          │
  │                   note="二维码暂未抓取成功")  │
  ├──────────────────────────────────────────────┤
  │  无 image_bytes 也无 entry_url               │
  │    → 跳过，不构卡                             │
  └──────────────────────────────────────────────┘
           ↓
  填充 OfficialGroup：
    platform ← 识别的平台（微信/QQ/飞书/Discord）
    group_type ← UNKNOWN（当前版本）
    source_urls ← 发现该入口的所有页面URL
    is_added ← False（用户未标记）
           ↓
  构卡返回 ProductCard
           ↓
  若 supported_groups 为空 → 该仓库不进入结果列表
```
