# 1. 页面布局

[引用：产品总纲 §7，§8]

### 1.1 整体结构

页面采用单页结构，自上而下分为搜索框、筛选栏、结果区三层，始终保持清晰的任务流顺序。

```text
┌──────────────────────────────────────────────┐
│ 搜索框                                        │
├──────────────────────────────────────────────┤
│ 筛选栏：平台 / 群类型 / 热门 / 最新 / 已添加开关 │
├──────────────────────────────────────────────┤
│ 结果区：主卡片列表 / 候选列表 / 空结果态 / 加载态 │
└──────────────────────────────────────────────┘
```

### 1.2 搜索框区域

| 属性 | 说明 |
|------|------|
| 位置 | 页面最顶部 |
| 输入内容 | 当前支持：产品名、英文别名 / 常见别名、官网域名 |
| 占位文案 | `搜索 AI 产品名称，如 Cursor、Claude、Lovable` |
| 触发方式 | 按 Enter、点击搜索按钮，或点击候选产品后触发正式搜索 |
| 候选态 | 存在歧义时，在搜索框下方弹出候选产品下拉列表，最多 3 条 |
| 清空操作 | 输入框有内容时显示清空按钮，点击后回到搜索前状态 |

### 1.2.1 未来入口预留：推荐关键词（后续迭代，当前未启用）
- 可在搜索框下方或侧边预留一组推荐词 chips。
- 推荐词属于探索型入口，不替代当前默认搜索输入。
- 点击推荐词后，进入“探索模式”：先扩展候选产品，再复用现有官方群发现流程。
- 当前设计阶段只预留位置和行为定义，不把推荐词 chips 视为已实现控件。
- 推荐词示例：`热门 AI 编程助手`、`开源 AI Agent`、`AI 视频生成`、`最近新增 AI IDE`

### 1.3 筛选栏区域

| 组成 | 说明 |
|------|------|
| 群平台 | 微信 / QQ / 飞书 |
| 群类型 | 交流群 / 答疑群 / 售后群 / 招募/内测群 |
| 排序按钮 | 热门 / 最新 |
| 状态开关 | 隐藏已添加 |

```text
┌──── 群平台 ────┐  ┌──────── 群类型 ────────┐  ┌── 排序 ──┐
│ 微信 │ QQ │ 飞书 │  │ 交流群 │ 答疑群 │ 售后群 │ 招募/内测群 │  │ 热门 │ 最新 │
└───────────────┘  └───────────────────────┘  └──────────┘
[ 隐藏已添加 ]
```

### 1.4 结果区

| 状态 | 展示方式 |
|------|----------|
| 搜索前 | 显示简洁引导态，提示用户输入产品名或域名 |
| 搜索中 | 显示骨架屏卡片，占位 2 至 3 个主卡片 |
| 搜索成功 | 显示主卡片列表 |
| 搜索歧义 | 显示候选产品下拉列表，由用户选择 |
| 未发现官方群 | 显示统一空结果态 |

补充说明：
- 当前“搜索前”状态不要求展示推荐关键词。
- 后续若启用推荐关键词，应作为搜索前的增强入口，而不是替换原有搜索框。
- 输入过程可用于更新候选产品提示，但不在仅输入时直接发起正式搜索。

# 2. 主卡片 / 展开子群

[引用：产品总纲 §6，§7]

### 2.1 主卡片（折叠态）

主卡片只展示产品级信息与一个默认显示的官方群预览，不引入 7 个字段之外的额外用户可见字段。

```text
┌──────────────────────────────────────────────┐
│ 应用名称                              [已添加] │
│ 应用简介                                      │
│ GitHub stars   创建时间   最近验证时间         │
│                                              │
│ [二维码缩略图 / 入口链接状态]   [群类型标签]    │
│ [查看入群方式]                                │
│                                    [展开查看]  │
└──────────────────────────────────────────────┘
```

### 2.2 折叠态字段展示规则

| 字段 / 控件 | 展示规则 |
|-------------|----------|
| 应用名称 | 位于卡片左上，作为主标题 |
| 应用简介 | 位于应用名称下方，一行或两行摘要展示 |
| GitHub stars | 与创建时间、最近验证时间放在同一信息行 |
| 创建时间 | 无值时显示“未知” |
| 最近验证时间 | 使用本次抓取成功时间，格式为 `YYYY-MM-DD HH:mm` |
| 入群方式 | 有二维码时显示缩略图；无二维码但有入口链接时显示状态提示和操作按钮 |
| 群类型 | 使用明确标签展示，仅允许 5 个枚举值 |
| 已添加按钮 | 放在卡片右上角，用于隐藏当前显示的群 |
| 展开查看 | 同一产品存在多个官方群时展示；只有一个官方群时不展示 |

### 2.3 主卡片（展开态）

展开态保留产品级信息区，并在下方按群逐条展示不同官方群入口。

```text
┌──────────────────────────────────────────────┐
│ 应用名称                              [已添加] │
│ 应用简介                                      │
│ GitHub stars   创建时间   最近验证时间         │
├──────────────────────────────────────────────┤
│ 群 1： [二维码缩略图 / 链接状态]  [交流群] [已添加] │
│ 群 2： [二维码缩略图 / 链接状态]  [答疑群] [已添加] │
│ 群 3： [二维码缩略图 / 链接状态] [招募/内测群] [已添加] │
│                                      [收起]    │
└──────────────────────────────────────────────┘
```

### 2.4 展开态规则
- 每一条子群只展示与该群相关的入群方式和群类型。
- “已添加”操作作用于当前子群，不影响同产品下其他子群。
- 展开态与折叠态共用同一组产品级字段，避免信息重复和字段漂移。

# 3. 二维码缩略图 / 放大查看

[引用：产品总纲 §6，§7]

### 3.1 缩略图展示

| 场景 | 展示方式 |
|------|----------|
| 有二维码 | 展示二维码缩略图，点击后进入放大查看 |
| 无二维码但有官方群入口链接 | 不展示缩略图，改为展示“二维码暂未抓取成功”说明和入群操作 |
| 没有二维码也没有官方群入口 | 该群不进入结果展示 |

### 3.2 放大查看（Modal）

```text
┌──────────────────────────────────────┐
│ 二维码大图                      [关闭] │
│                                      │
│            [二维码放大图]             │
│                                      │
│ 应用名称                              │
│ 群类型：交流群                         │
│ 最近验证时间：2026-04-01 15:20         │
│                                      │
│              [进入官方群]              │
└──────────────────────────────────────┘
```

### 3.3 交互规则
- 点击二维码缩略图：打开二维码放大查看。
- 点击“查看入群方式”：
  - 有二维码时，打开二维码放大查看。
  - 无二维码但有链接时，直接打开官方群入口链接。
- 当展示的是入口链接而非二维码时，文案必须同时提示“二维码暂未抓取成功”。

# 4. 空结果态 / 已隐藏结果态

[引用：产品总纲 §7]

### 4.1 空结果态

```text
┌──────────────────────────────────────────────┐
│ 未发现该产品的官方群                           │
│                                              │
│ 未在官网或 GitHub 官方页面找到官方群入口。      │
│ 你可以尝试：                                   │
│ 1. 更换产品名或别名                            │
│ 2. 直接输入官网域名                            │
└──────────────────────────────────────────────┘
```

空结果态文案规则：
- 统一使用“未发现该产品的官方群”或“未发现官方群”。
- 不展示“联系我们”、公众号二维码、疑似社区等替代入口。
- 不展示技术异常细节或抓取失败堆栈信息。

### 4.2 已隐藏结果态

```text
┌──────────────────────────────────────────────┐
│ [隐藏已添加：开启]                            │
├──────────────────────────────────────────────┤
│ 开启时：已添加的群不展示                       │
│ 关闭时：已添加的群以灰色弱化样式展示            │
└──────────────────────────────────────────────┘
```

已隐藏结果态规则：
- 默认开启“隐藏已添加”。
- 关闭开关后，已添加群重新出现在原结果列表中，但使用灰色弱化样式，便于区分。
- 已添加群保留原字段信息，不修改字段值。

### 4.3 加载态
- 使用骨架屏，而不是单独的转圈加载。
- 骨架屏结构与主卡片结构一致，减少布局跳动。
- 搜索结果返回后，骨架屏原位替换为真实内容。

### 4.4 候选产品态
- 候选列表显示在搜索框下方，不占用主结果区的长期布局。
- 每个候选项展示：应用名称、简短描述、来源域名。
- 点击候选项后，进入正式搜索并刷新结果区。

# 5. 组件清单与状态汇总

### 5.1 组件清单

| 组件名 | 作用 | 关键状态 |
|--------|------|----------|
| SearchBox | 输入搜索词并触发搜索 | 默认 / 输入中 / 候选中 |
| CandidateDropdown | 展示候选产品 | 展开 / 收起 / 已选择 |
| FilterBar | 展示平台、群类型、排序与状态开关 | 默认 / 已筛选 |
| ProductCard | 展示产品主卡片 | 折叠 / 展开 / 已弱化 |
| GroupRow | 展示展开后的单个官方群条目 | 默认 / 已添加 |
| QRModal | 展示二维码放大查看 | 打开 / 关闭 |
| EmptyState | 展示空结果态 | 单一状态 |
| SkeletonCard | 展示加载占位 | 加载中 |

### 5.2 交互状态汇总

| 用户动作 | 系统反馈 |
|----------|----------|
| 输入搜索词 | 可触发正式搜索，或在歧义场景下展示候选列表 |
| 点击候选产品 | 切换到该产品的正式搜索结果 |
| 点击“热门” | 当前结果按 GitHub stars 重新排序 |
| 点击“最新” | 当前结果按最近验证时间重新排序 |
| 点击“已添加” | 当前群从默认结果中隐藏 |
| 关闭“隐藏已添加” | 已隐藏群回到列表，并以弱化样式展示 |
| 点击二维码缩略图 | 打开二维码放大查看 |
| 点击入口链接 | 直接跳转到官方群入口 |

未来入口预留：
- 若后续增加推荐关键词，交互应为“点击推荐词 → 进入探索模式 → 输出现有主卡片列表”。
- 不把推荐关键词设计成“输入泛词后直接全网搜二维码”的交互。
