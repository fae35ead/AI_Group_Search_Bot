## 项目背景
快速、高效地加入尽可能多的热门AI工具群聊，以便于后期进行公司产品的宣传。

## 项目核心难点
- 如何返回**准确**的二维码和群聊链接
- 如何返回**尽可能多**的二维码和群聊链接
- 如何在保证返回数量的基础上，优化搜索返回响应**速度**
- 如何快速、多量地加入群聊，如实现批量**自动**加群、全自动代理加群等方式

## 已实现功能
- **搜索**
	- 手动搜索：输入关键词后，返回对应结果。
		- 返回的结果是两列的群聊卡片，若有二维码，会在卡片右端优先显示二维码；若没有则显示链接。
		- 微信/钉钉/飞书基本以二维码形式出现；QQ/Discord基本以链接形式出现；QQ还以QQ群号的形式出现。
		- 可以展开返回卡片，记录同一个产品下更多的入群方式。

	![[Pasted image 20260410134125.png]]
	
	- ”全自动搜索“：点击按钮后，自动选择”今日推荐“中的词进行搜索，搜索后可以一键入库，入库后自动进行下一轮搜索。
	![[Pasted image 20260410135450.png]]
	
	- 搜索链路：先抓取 GitHub/官网候选页；命中社群信号后提取二维码、链接或 QQ 群号；结果不足时再扩展相关子页，并在必要时用 Playwright 补抓动态内容，最后去重生成结果卡片。
	- 提速机制：
		- 若已搜索过相同关键词，会直接返回上一次搜索的结果。点击”验证“按钮会重新爬取页面信息，防止二维码更新但未被记录。
		- 使用异步搜索，后端创建Job的机制。
			- 当返回结果数量很大时，会优先返回10个结果，后续会再次补充。
			- Job会继续搜索尽量多的结果，并陆续返回。
			- Job可能会搜索超出设定的返回结果，这是为了便于后续拉高返回结果数量时快速出现结果。

- **筛选**
	- 搜索栏筛选：支持筛选Github星级、创建时间、群聊平台、返回数量
	
	![[Pasted image 20260410134738.png]]
	
	- 库内筛选：支持筛选群聊平台、是否入群、是否忽略；支持根据名称搜索
	
	![[Pasted image 20260410134818.png]]
		- 忽略：点击忽略，该群聊卡片则不再出现在搜索返回结果且不再出现在库内，但仍然可通过筛选查看并恢复。
	
- **推荐词系统**
	- 调用Github API选取热门词汇，以星级为指标
	- 30分钟自动切换一批新词汇，也支持手动切换
	- 已发现的推荐词会记录本地，避免每一次都重新调用API查找导致响应时间慢的问题

![[Pasted image 20260410134837.png]]

- **入库**
	- 对搜索返回的群聊卡片点击入库后，卡片不会再此出现在返回结果中。
	- 入库后的卡片不随着搜索状态更新，相当于本地离线化了。如果需要拉取更新信息，需要出库后再次验证/搜索。
	- 库内的卡片可以标记已入群/未入群，便于后续区分。
